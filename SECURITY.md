# Security Policy

## Supported Versions

The following versions of this project are currently being supported with security updates:

| Version | Supported |
| ------- | --------- |
| 1.x.x   | ✅ Yes     |
| < 1.0   | ❌ No      |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### 📩 How to report

* Email: [223002089@student.green.ac.bd](mailto:223002089@student.green.ac.bd)
* Or open a private GitHub security advisory (preferred)

### ⚠️ Please DO NOT:

* Open a public issue for security vulnerabilities
* Disclose the vulnerability publicly before it is fixed

---

## What to include in your report

Please provide:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Suggested fix (if possible)

---

## Response Time

We aim to respond within **48–72 hours** and provide a fix as soon as possible.

---

## Security Best Practices

We take security seriously and follow these practices:

* Input validation for all user data
* Secure API usage and environment variables
* No storage of sensitive personal data
* Regular dependency updates

---

## Scope

This policy applies to:

* Backend APIs (FastAPI services)
* AI processing modules
* Resume parsing components

---

## Acknowledgements

We appreciate responsible disclosure and thank contributors who help improve the security of this project.
